//overview

A server-authoritative round and ability system designed for flexibility, security, and ease of expansion. Supports modular abilities with per-ability cooldowns, antispam protection, and a debug mode on both server and client to disable cooldowns and round checks during development.

//features
Security
All abilities are executed server-side. Client input is validated before any action runs. Cooldowns and round checks enforce against remote spam and exploit abuse.
Modularity
Each ability is a self-contained ModuleScript. The system loads all abilities automatically. Add new abilities without touching core logic.
Debug Mode
Both the server and client scripts expose a debug mode flag that disables cooldowns and round checks. Flip it off before shipping.

//configuration
Round timing
Open GameState in ServerScriptService/Modules to adjust the two timing values at the top of the file. These are the only values you need to change to control how long each phase lasts.
GameState.RoundTime
Duration of each round in seconds. Default is 120.
GameState.Intermission
Duration of the lobby waiting period between rounds in seconds. Default is 15.
//gamestate
GameState is a lightweight module that tracks the current phase of the game and stores round timing settings. It is the single source of truth for round status, all other scripts read from it rather than tracking state themselves. Configure round length and intermission duration here.
GameState.RoundTime
Duration of a round in seconds. Default is 120. Change this to control how long each round lasts.
GameState.Intermission
Duration of intermission in seconds. Default is 15. This is the gap between rounds where players wait in the lobby.
GameState.Status
The current game phase. Possible values are "Waiting", "Intermission", "Round", and "Ending". Read this from any server script to check what phase the game is in.
GameState:SetStatus(status)
Sets the current game phase. Called internally by RoundManager. You can call this from your own scripts to force a phase change, for example when a win condition is met.
GameState:GetStatus()
Returns the current phase string. Use this to gate logic in your own scripts. For example, only allow certain actions during "Round".
GameState:GetRoundTime()
Returns the configured round duration. RoundManager uses this to run the countdown.
GameState:GetIntermission()
Returns the configured intermission duration. RoundManager uses this to run the lobby countdown.
//roundmanager
RoundManager is the main game loop script. It drives the intermission, round, and end phases in sequence, fires round status updates to all clients via RoundStatus, and handles all incoming ability requests from players. It also enforces a per-player request lock to prevent simultaneous ability spam.
RoundManager:StartIntermission()
Sets GameState to "Intermission" and runs a countdown from GameState.Intermission to 0, firing RoundStatus to all clients each second. Yields until complete.
RoundManager:StartRound()
Sets GameState to "Round" and runs a countdown from GameState.RoundTime to 0, firing RoundStatus to all clients each second. Yields until complete.
RoundManager:EndRound()
Sets GameState to "Ending", fires a RoundEnded event to all clients, then waits 5 seconds before the loop continues to intermission.
AbilityEvent.OnServerEvent
Listens for ability requests from clients. Validates the ability name type, applies the per-player request lock, calls AbilityManager:UseAbility(), then fires the result back to the requesting client.
Minimum player check
The main loop checks PlayerManager:GetPlayers() after each intermission. If fewer than 1 player is connected, the round is skipped and the loop waits 3 seconds before trying again. Change the threshold to suit your game.
//integrating with your own scripts
Checking the current game phase
Require GameState from any server script to read the current phase and gate your own logic accordingly.
local GameState = require(game.ServerScriptService.Modules.GameState) -- only allow something during an active round if GameState:GetStatus() == "Round" then -- your logic here end
Ending a round early
If your game has a win condition - last player standing, score threshold, objective complete. You can force the round to end by setting the status directly. RoundManager's countdown loop checks the status on each tick and will exit when it detects "Ending".
local GameState = require(game.ServerScriptService.Modules.GameState) local Remotes = game.ReplicatedStorage.Remotes -- call this from any server script when the win condition is met local function endRoundEarly(winner) GameState:SetStatus("Ending") Remotes.RoundStatus:FireAllClients("RoundEnded", winner) end
▸
To make RoundManager actually respect an early end, add a status check inside the StartRound countdown loop so it breaks when GameState is no longer "Round".
Listening for round events on the client
RoundStatus fires to all clients every second during intermission and the round, and once on round end. Connect to it in any LocalScript to update your UI or trigger client-side effects.
local RoundStatus = game.ReplicatedStorage.Remotes.RoundStatus RoundStatus.OnClientEvent:Connect(function(phase, timeRemaining) if phase == "Intermission" then -- update lobby timer UI elseif phase == "Round" then -- update round timer UI elseif phase == "RoundEnded" then -- show end screen, play sound, etc. end end)
Triggering an ability from your own UI
AbilityClient handles keyboard input by default. If you want to trigger an ability from a GUI button instead, fire AbilityEvent directly from your own LocalScript.
local AbilityEvent = game.ReplicatedStorage.Remotes.AbilityEvent -- fire from a button click or any client-side trigger myButton.MouseButton1Click:Connect(function() AbilityEvent:FireServer("Dash", {}) end) -- listen for the result fired back from the server AbilityEvent.OnClientEvent:Connect(function(success, abilityName) if success then -- play animation, update cooldown UI, etc. end end)
Running code at the start and end of each phase
The cleanest way to hook into phase transitions without editing RoundManager is to poll GameState from a separate script, or extend RoundManager's methods directly to call your own functions at the right moment.
-- in RoundManager, add your hooks at the top of each method function RoundManager:StartRound() GameState:SetStatus("Round") -- spawn enemies, reset map, distribute weapons, etc. onRoundStart() for i = GameState:GetRoundTime(), 0, -1 do RoundEvent:FireAllClients("Round", i) task.wait(1) end end function RoundManager:EndRound() GameState:SetStatus("Ending") -- tally scores, announce winner, clean up map, etc. onRoundEnd() RoundEvent:FireAllClients("RoundEnded") task.wait(5) end

//creating a new ability
Create a new ModuleScript inside Modules > Abilities and name it your ability. The system loads it automatically.
-- Modules/Abilities/Fireball local Ability = {} Ability.Cooldown = 5 function Ability:Use(player, data) -- your ability logic here print(player.Name .. " used Fireball") return true end return Ability

//tips and security
▸
Always return true if the ability succeeds and return false if it fails
▸
Keep all ability logic inside the module and never trust or rely on client-sent data
▸
Server cooldowns are the authority and client cooldown is for UI responsiveness only
▸
Use GameState:GetStatus() inside your own ability logic to prevent abilities firing outside of a round
▸
Disable debug mode on both server and client before going live
