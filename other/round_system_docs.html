Ability System
Server-authoritative round management · Modular abilities · Anti-exploit protection
//overview

A server-authoritative round and ability system designed for flexibility, security, and ease of expansion. Supports modular abilities with per-ability cooldowns, antispam protection, and a debug mode on both server and client to disable cooldowns and round checks during development.

//features
Security
All abilities are executed server-side. Client input is validated before any action runs. Cooldowns and round checks enforce against remote spam and exploit abuse.
Modularity
Each ability is a self-contained ModuleScript. The system loads all abilities automatically. Add new abilities without touching core logic.
Debug Mode
Both the server and client scripts expose a debug mode flag that disables cooldowns and round checks. Flip it off before shipping.
//system flow
Every ability request travels through this pipeline. The server is the final authority at every step.
Client input
→
AbilityEvent remote
→
AbilityManager validation
→
Ability execution
→
Result to client

//creating a new ability
Create a new ModuleScript inside Modules > Abilities and name it your ability. The system loads it automatically.
-- Modules/Abilities/Fireball local Ability = {} Ability.Cooldown = 5 function Ability:Use(player, data) -- your ability logic here print(player.Name .. " used Fireball") return true end return Ability
//client setup
Abilities are defined on the client with a key binding and local cooldown for responsiveness. The server is the final authority and local cooldown is display only.
Dash = { Key = Enum.KeyCode.Q, Cooldown = 10, LastUse = 0 }
01
Player presses the bound key
02
Client fires a request to the server via AbilityEvent
03
Server validates cooldown, round state, and spam protection
04
Ability executes if all checks pass and the result is returned to the client
//included abilities
Dash
Moves the player forward quickly using direction input. Short-duration movement burst with its own cooldown.
BigHead
Temporarily increases the player's head size. Automatically resets after a few seconds.
//tips and security
▸
Always return true if the ability succeeds and return false if it fails
▸
Keep all ability logic inside the module and never trust or rely on client-sent data
▸
Server cooldowns are the authority and client cooldown is for UI responsiveness only
▸
Disable debug mode on both server and client before going live
//troubleshooting
?
Ability not firing - check that AbilityEvent and RoundStatus RemoteEvents exist in ReplicatedStorage/Remotes
?
Cooldown not working - confirm Cooldown is set inside the ability module
?
Nothing happens - check the output for errors and verify all scripts are in the correct locations
